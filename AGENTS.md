# 项目协作说明

## 项目概览

这是一个纯前端的手势粒子互动页面，入口为 `index.html`。页面使用全屏 `<canvas>` 渲染动态粒子效果，支持鼠标/触控交互、图片上传粒子化，以及通过 MediaPipe Hands 调用摄像头识别手势。

项目没有构建系统、包管理配置或本地依赖安装步骤，当前文件可以直接在浏览器中打开运行。不过手势识别依赖 CDN 和摄像头权限，通常更适合通过本地静态服务器访问。

## 文件结构

- `index.html`：页面结构、Canvas、工具栏、模式按钮、摄像头预览元素，以及外部 MediaPipe 脚本引用。
- `styles.css`：全屏舞台、顶部工具栏、模式按钮、读数标签、摄像头预览和响应式布局样式。
- `app.js`：粒子系统、图片采样、动画循环、鼠标/触控输入、MediaPipe Hands 手势识别和模式切换逻辑。
- `.gitignore`：忽略 `.agents/`、`.codex/`、依赖目录、构建产物、缓存、日志和环境变量文件。

## 运行方式

可以直接打开 `index.html` 预览基础粒子交互：

```text
index.html
```

如果需要摄像头和 CDN 脚本稳定工作，建议在项目根目录启动一个静态服务器，例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 核心交互

- 鼠标或触控按下、拖动时，会在指针位置持续发射当前模式的粒子。
- 点击画布时会额外触发水波扩散效果。
- 上传图片后，图片会被采样成彩色粒子，并在画布中央重组。
- 顶部按钮可手动切换 5 种模式：彩雾、火焰、水流、星河、泡泡。
- 开启摄像头后，MediaPipe Hands 会读取手部关键点：
  - 食指位置作为粒子发射位置。
  - 伸出的手指数用于切换粒子模式。
  - 手掌压力估算值较高时触发水波扩散。

## JavaScript 结构

`app.js` 的主要职责如下：

- 初始化 DOM 引用和 Canvas 2D 上下文。
- 用 `MODES` 定义每种模式的显示标签与色相范围。
- 用 `particles` 管理临时发射粒子，用 `imageParticles` 管理图片重组粒子，用 `ripples` 管理水波。
- `resize()` 处理 DPR、画布尺寸和图片粒子重建。
- `setMode()` 更新当前模式、按钮高亮和读数标签。
- `spawnParticle()` / `spawnBurst()` 根据模式生成粒子初始速度、大小、寿命和颜色。
- `drawParticle()` / `updateParticles()` 负责各模式粒子的绘制与生命周期更新。
- `buildImageParticles()` 将上传图片缩放采样为粒子目标点。
- `drawImageParticles()` 让图片粒子回到目标位置，并被手势或鼠标推开。
- `drawBackground()` 和 `animate()` 驱动持续动画。
- `startCamera()` 初始化 MediaPipe Hands 与摄像头流。
- `countExtendedFingers()` 根据关键点估算伸出手指数。
- `palmPressure()` 根据掌宽和掌高估算触发水波的压力值。

## 外部依赖

页面通过 CDN 引入：

- `@mediapipe/camera_utils`
- `@mediapipe/hands`

这些依赖只在浏览器运行时加载，不存在本地 `node_modules`。如果网络不可用，鼠标和触控交互仍可使用，但手势识别不可用。

## 样式与设计约定

- 页面是全屏沉浸式 Canvas 应用，`body` 禁止滚动。
- 工具栏固定在顶部，读数标签固定在左下角，摄像头预览固定在右下角。
- UI 使用深色半透明面板、细边框、模糊背景和彩色高亮。
- 响应式断点在 `820px` 和 `520px`，窄屏下工具栏改为更紧凑的两列布局。
- 按钮圆角较小，整体偏工具型界面，不是营销落地页。

## 已知注意事项

- 当前 `index.html` 和 `app.js` 中的中文文案存在乱码，例如标题、按钮、状态提示和模式标签。这很可能是历史编码转换问题。修改文案时建议统一保存为 UTF-8。
- `index.html` 中部分乱码导致标签文本看起来异常，维护时需要检查 HTML 是否仍保持合法结构。
- 摄像头权限通常要求安全上下文；`localhost` 通常可用，直接用 `file://` 打开时可能受浏览器限制。
- `app.js` 里有粒子数量上限：普通粒子超过 1300 会裁剪旧粒子，图片粒子采样目标大约为 `16200` 个点。
- 手势模式映射由 `modeByFingerCount` 控制：0/1 指为彩雾，2 指火焰，3 指水流，4 指星河，5 指泡泡。

## 修改建议

- 优先修复乱码文案，确保页面中文显示正常。
- 如果继续扩展交互，尽量沿用现有模式结构：在 `MODES`、`spawnParticle()`、`drawParticle()`、`updateParticles()` 中补齐对应分支。
- 如果增加新依赖，先判断是否需要构建系统；当前项目保持零构建、静态文件直开。
- 调整 Canvas 性能时，重点关注 `IMAGE_PARTICLE_DETAIL`、普通粒子上限和每帧绘制逻辑。
- 改动手势逻辑后，建议同时验证无摄像头、摄像头不可用、授权成功三种路径。

## 验证清单

- 页面能加载并显示全屏粒子背景。
- 鼠标按下和拖动可以持续生成粒子。
- 点击画布可以触发水波。
- 模式按钮可以切换效果并更新高亮。
- 上传图片后，画面中央出现图片粒子化效果。
- 在可用摄像头和网络环境下，点击开启手势后可以看到摄像头预览并通过手指数切换模式。
